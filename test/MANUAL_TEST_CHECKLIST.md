# Add to Toodledo — 手動テストチェックリスト

## 前提条件
- Chrome に拡張機能をインストール済み
- テスト用のToodledoアカウントを用意（本番アカウントでもOK、テストタスクは後で削除）

---

## A. 段階1機能（Web版で追加）

### A-1. 基本動作
- [ ] 適当なWebページ（例: https://ja.wikipedia.org ）を開く
- [ ] ツールバーのアイコンをクリック → ポップアップが表示される
- [ ] タスク名にページタイトルが自動入力されている
- [ ] ノートにページURLが自動入力されている

### A-2. 選択テキスト反映
- [ ] ページ上でテキストを選択した状態でアイコンをクリック
- [ ] タスク名に選択テキストが入っている
- [ ] ノートにURL + `[ページタイトル]` が入っている

### A-3. Web版追加モード
- [ ] 「Web版で追加」タブに切り替える
- [ ] 「Toodledo で追加」ボタンをクリック
- [ ] Toodledoの画面が新しいタブで開く
- [ ] クリップボードにタスク名がコピーされている（Cmd+V で確認）

### A-4. コピー機能
- [ ] 「コピー」ボタンをクリック
- [ ] クリップボードにタスク名+ノートがコピーされている

### A-5. 特殊ページ対応
- [ ] `chrome://extensions/` でアイコンをクリック → エラーなくポップアップが表示される
- [ ] 新しいタブ（`chrome://newtab/`）でアイコンクリック → エラーなく動作

---

## B. 段階2機能（API連携）

### B-1. 設定画面
- [ ] ポップアップ右上の「設定」リンクをクリック → 設定画面が開く
- [ ] Redirect URI が表示されている（`https://<extension-id>.chromiumapp.org/` 形式）
- [ ] Redirect URI をクリック → クリップボードにコピーされる

### B-2. API設定の保存
- [ ] Client ID と Client Secret を入力
- [ ] 「設定を保存」ボタンをクリック → 「設定を保存しました」メッセージが表示
- [ ] ページをリロード → 入力値が復元されている

### B-3. Toodledo連携（OAuth認証）
- [ ] 「Toodledoと連携」ボタンをクリック
- [ ] Toodledoの認証ウィンドウが開く
- [ ] Toodledoにサインインし、アプリを許可
- [ ] 「Toodledoと連携しました」メッセージが表示される
- [ ] 接続状態が「接続済み」に変わる

### B-4. 接続テスト
- [ ] 「接続テスト」ボタンをクリック
- [ ] 「接続成功」メッセージとアカウント情報が表示される

### B-5. APIモードでタスク追加
- [ ] 適当なWebページを開く
- [ ] ポップアップを開き、「API で追加」モードであることを確認
- [ ] タスク名を編集（任意）
- [ ] 「タスクを追加」ボタンをクリック
- [ ] 「タスク〇〇を追加しました！（ID: xxx）」メッセージが表示される
- [ ] Chrome通知が表示される
- [ ] Toodledo Web版で当該タスクが追加されていることを確認
- [ ] ノート欄にURLが記録されている
- [ ] タグが設定されている（デフォルトタグを設定している場合）

### B-6. エラーハンドリング
- [ ] タスク名を空にして「タスクを追加」→ バリデーションエラーが表示される
- [ ] 未認証状態で「タスクを追加」→ 適切なエラーメッセージが表示される

### B-7. ログアウト
- [ ] 設定画面で「ログアウト」ボタンをクリック
- [ ] 確認ダイアログが表示される → OK
- [ ] 接続状態が「未接続」に戻る

## C. 自動テスト（API単体テスト）

### C-1. 実行手順
```bash
# 1. 環境変数をセット
export TOODLEDO_ACCESS_TOKEN="<access_tokenの値>"

# 2. テスト実行
node test/api-unit-test.mjs
```

### C-2. access_token の取得方法
1. Chrome で拡張機能と連携済みの状態にする
2. `chrome://extensions/` → Add to Toodledo → 「Service Worker」リンクをクリック
3. DevTools Console で以下を実行:
```javascript
chrome.storage.local.get(['accessToken'], (d) => console.log(d.accessToken));
```
4. 表示されたトークンを環境変数にセット

### C-3. 期待結果
- テスト1（アカウント取得）: ✅ すべてパス
- テスト2（タスク追加）: ✅ すべてパス、テストタスクがToodledoに追加される
- テスト3（エラーケース）: ✅ エラーコード601が返される

## D. 補足: 既知の制限事項

- Toodledo の Quick Add URL にはパラメータでタスク名を直接渡す機能がないため、Web版追加モードではクリップボードコピー+手動貼り付けが必要
- Access Token の有効期限は2時間。期限切れ時は自動的にRefresh Tokenで更新される
- Refresh Token の有効期限は30日。30日以上利用しない場合は再認証が必要
- `chrome://` や `edge://` など特殊ページではcontent scriptが動作しないため、選択テキスト取得は不可
